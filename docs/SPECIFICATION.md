# 📋 SPECIFICATION.md

## 1. Core Functional Requirements

### 1.1 Authentication & Profile
- **Password Auth**: Argon2id hashing for local credentials.
- **OTP**: Email-based verification for login/sensitive actions.
- **GitHub OAuth**: Single-click registration and linking.
- **Profile**: Manage public info and security preferences.

### 1.2 VPS Management
- **Connection**: SSH-key based authentication (generated by DeployForge or provided by user).
- **Discovery**: Automatically detect Docker, Nginx, and OS details upon connection.
- **Provisioning**: Optional "Setup Server" command to install Docker/Nginx dependencies.
- **Health**: Real-time status of the VPS (Online/Offline).

### 1.3 Deployment System
- **Framework Detection**: Support for:
  - **Next.js**: Hybrid, Static, or Node SSR.
  - **React/Vite/Vue**: Static build + Nginx serving.
  - **Node.js/Express/Fastify**: Standard containerization.
  - **Python/Django/Flask**: Gunicorn/Uvicorn setups.
  - **PHP/Laravel**: Artisan/Octane configurations.
- **Build Types**:
  - **Dockerfile**: Use existing repository Dockerfile.
  - **Nixpacks/Buildpacks**: Automated detection and build.
- **Port Management**: Automatic port allocation (starting from 3000) with conflict detection.
- **Environment Variables**: Encrypted secret management per application.

### 1.4 Domain & Networking
- **Auto-Nginx**: Generation of `sites-available` files.
- **SSL**: Automatic `Certbot` integration for Let's Encrypt certificates.
- **Subdomains**: Support for system-generated and custom subdomains.

---

## 2. Technical Requirements

### 2.1 Backend Performance
- Standard response time < 200ms.
- Asynchronous task processing for all long-running operations (> 2s).

### 2.2 Security Standards
- All VPS SSH keys stored with AES-256-GCM.
- GitHub Personal Access Tokens (PATs) and OAuth tokens encrypted.
- No plaintext secrets in logs.

### 2.3 Limits (Free/Pro Tiers)
- Maximum simultaneous builds.
- Retention period for deployment logs (e.g., 7 days).
- Number of managed VPS instances.

---

## 3. Deployment Sandbox Features
- **Build Validation**: Verify `package.json` or equivalent before starting build.
- **Resource Check**: Pre-scan VPS for available RAM/CPU before deploying.
- **Security Validation**: Scan for common vulnerabilities in base images.
- **Readiness Score**: A 0-100 score indicating if the app is ready for production.
